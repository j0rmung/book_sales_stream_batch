# Пакетная и потоковая обработка данных

## Цель работы

Закрепить навыки работы с потоками данных, реализовав два типа обработки (потоковую и пакетную) для одного набора данных с использованием брокера сообщений Apache Kafka и фреймворка Flask.

## Описание проекта

В работе реализована система сбора и аналитики событий о продажах книг в реальном времени и в режиме отложенной агрегации:

* **Flask API (Producer)** — принимает события о продажах через HTTP POST-запросы, отправляет их в топик Kafka для потоковой обработки и параллельно логирует в локальный файл для пакетной обработки.
* **Stream Processing Service** — непрерывно читает поток данных из Kafka и рассчитывает скользящее среднее стоимости продаж за последние 5 минут.
* **Batch Processing Service** — имитирует ежедневный аналитический джоб (запускаемый по расписанию), который рассчитывает общую выручку и формирует топ-5 самых продаваемых книг на основе накопленного массива данных.

## Используемые технологии

* Python 3
* Flask
* Apache Kafka & Apache Zookeeper
* Docker & Docker Compose
* kafka-python
* pandas

## Конфигурация брокера Apache Kafka

В проекте используется топик:

* `book_sales`

Топик инициализируется автоматически при отправке первого сообщения Flask-сервисом или при первом подключении подписчика. Сообщения передаются в формате JSON с сериализацией в байты `utf-8`.

## Установка и запуск

### 1. Запуск инфраструктуры (Kafka + Zookeeper)

Перейдите в корневую директорию проекта и запустите контейнеры в фоновом режиме:

```bash
docker compose up -d

```

После запуска брокер будет доступен по адресу: `localhost:9092`.

### 2. Подготовка окружения

Создайте виртуальное окружение и установите необходимые библиотеки:

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

```

### 3. Запуск потоковой обработки

В отдельном окне терминала запустите бесконечный процесс обработки потока:

```bash
source venv/bin/activate
python stream_job.py

```

### 4. Запуск Flask-приложения

В еще одном окне терминала запустите веб-сервер для приема транзакций:

```bash
source venv/bin/activate
python app.py

```

## Проверка работы

### 1. Генерация потока событий

Отправьте несколько тестовых POST-запросов, имитирующих продажи книг:

```bash
curl -X POST http://127.0.0.1:5000/sale \
-H "Content-Type: application/json" \
-d "{\"book_id\": \"101\", \"title\": \"Grokking Algorithms\", \"price\": 35.50}"

curl -X POST http://127.0.0.1:5000/sale \
-H "Content-Type: application/json" \
-d "{\"book_id\": \"102\", \"title\": \"Designing Data-Intensive Applications\", \"price\": 45.00}"

```

*В этот момент в терминале со `stream_job.py` мгновенно отобразится перерасчет скользящего среднего за последние 5 минут.*

### 2. Запуск пакетного аналитического отчета

Имитируйте наступление конца дня и запустите скрипт пакетной агрегации накопленных логов:

```bash
python batch_job.py

```

*Скрипт выведет итоговую выручку и топ-5 книг, после чего завершит выполнение.*

## Сравнительный анализ подходов

| Критерий | Потоковая обработка (Stream) | Пакетная обработка (Batch) |
| --- | --- | --- |
| **Латентность** | Минимальная (Near Real-Time). Данные обрабатываются сразу. | Высокая. Обработка запускается по расписанию (раз в сутки). |
| **Управление состоянием** | Сложное. Состояние скользящего окна удерживается и обновляется в памяти. | Простое. Данные статичны, состояние между запусками хранить не требуется. |
| **Объем данных** | Позаписный. Работает со свежими событиями внутри временного окна. | Монолитный. Обрабатывает весь массив накопленных данных за период целиком. |
| **Бизнес-цель** | Оперативный мониторинг трендов и аномалий "прямо сейчас". | Формирование точной финансовой отчетности и долгосрочной аналитики. |
